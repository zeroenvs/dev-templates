# Philosophy

ZeroEnvs is built around a simple idea:

> Development environments should not require manual setup.

---

## Core Beliefs

### 1. Zero-Install by Default

Developers should not need to install SDKs, runtimes, or tools locally.

A working environment must be available with minimal effort using containers.

---

### 2. Environments Are Disposable

Development environments should be:

* easy to create
* easy to destroy
* safe to reset

---

### 3. Reproducibility Matters

The same environment should behave identically across machines.

No “works on my machine” scenarios.

---

### 4. Infrastructure Is Part of the Project

Environment setup is not a side concern.

It is part of the development experience and must be versioned and shared.

---

### 5. Simplicity Over Standardization

We prioritize:

* simple onboarding
* clear structure

Over:

* rigid command standards
* over-engineered abstractions

---

### 6. Templates Are Autonomous

Each template is free to define:

* its workflow
* its tooling
* its complexity

As long as it follows the structural contract.

---

### 7. Developer Experience First

The system should reduce friction, not introduce new layers of complexity.

---

## Goal

To create an open ecosystem where development environments are:

* easy to use
* easy to share
* easy to reproduce

---

## Non-Goals

* Standardizing how developers write code
* Enforcing strict workflows
* Replacing existing development tools

---

## Evolution

This philosophy may evolve as the ecosystem grows, but the core principles should remain stable.
